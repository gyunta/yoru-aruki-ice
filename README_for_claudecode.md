## claude.aiで作成済みのファイル

### 1. 事業計画書 (`事業計画書_夜散歩アイス.md`)
以下のセクションで構成された事業計画書。

- 事業コンセプト
- 事業の背景・動機

- ロードマップ（Phase 0〜3）

### 2. PLシミュレーター（インタラクティブHTML）
claude.ai上でウィジェットとして動作中。以下のパラメーターをスライダーで操作できる。

**入力パラメーター：**
- 営業回数（月）：1〜12回
- 1回あたり販売数：5〜50個
- 客単価：400〜1,000円
- 原価率：20〜60%
- 出店料（1回）：0〜8,000円
- 消耗品費（1回）：0〜2,000円

**出力：**
- 月間売上・コスト・粗利
- 損益分岐点（何回営業すれば黒字か）
- 営業回数×月間粗利の棒グラフ

## Claude Codeへの依頼事項

### Step 1: フォルダ構成の作成
以下の構成でローカルフォルダを作成してください。

```
yoru-aruki-ice/
├── README.md                   ← このファイル
├── business-plan/
│   ├── 事業計画書_夜散歩アイス.md
│   └── pl-simulator.html       ← Step 2で作成
└── web/                        ← Step 3以降で作成
    ├── index.html
    ├── simulator.html
    └── style.css
```

### Step 2: PLシミュレーターのHTML化
claude.aiで作成したシミュレーター（上記仕様）を `business-plan/pl-simulator.html` として実装してください。

**要件：**
- スタンドアロンで動作するHTML単一ファイル
- スライダーで全パラメーターを操作可能
- 月間損益・損益分岐点をリアルタイム表示
- 棒グラフで1〜12回の損益推移を表示（Chart.js使用）
- レスポンシブ対応

### Step 3: GitHubリポジトリのセットアップ
- リポジトリ名: `yoru-aruki-ice`
- GitHub Pagesを有効化（`main`ブランチ、`/root`または`/web`フォルダ）
- PLシミュレーターをWeb公開

### Step 4: 将来の更新フロー（仕組みづくり）
数字が精緻化されるたびに以下のフローで更新できるようにする。

```
① オーナーがClaude Codeに「原価率を40%に変更して」と指示
② Claude Codeがpl-simulator.htmlを直接編集
③ git add → commit → push
④ GitHub Pagesに自動反映
```
